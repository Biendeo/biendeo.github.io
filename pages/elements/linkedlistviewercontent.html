The Linked List Viewer!

In my COMP1917 course, I learned about linked lists, which are basically an example of dynamic memory creation. There are no hard-coded limits when dealing with a linked list, as each element in the list just ties to the next one. It's a bit inconvenient as there's no easy index jump to get a specific element. However, you can always make a function that does that.

The most annoying part about linked lists is that you have to watch out for the end of a list, as they're null objects. This means that you can't dereference them for data, or else it'll crash the program. So, you have to add cases to everything, which is a pain. Honestly, it's not too bad, but it gets you thinking about how literal computers are.

The linked list viewer I made basically recreates the whole menu system from Terminal Wars, and applies it to linked lists. It displays the linked list, and allows the user to add, edit, and delete terms. They're handled by a linked list, so there's no hard-coded limit on how many terms there can be. It also means that smaller lists load quicker than larger ones.

I had originally planned from the program to save this list to a file (similar to Terminal Wars), but I never got around to really doing it. I feel as if I had already done the basic implementation, and it was enough to talk about it as work for that course.
